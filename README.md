# 7-Day Learner Challenge: Tiny Tamagotchi MVP

Submission template for the **Tiny Tamagotchi MVP with Spec-Driven Development** challenge. Fork or clone this repo, fill in the specs and features, and submit.

## Structure

```
specs/
  mission.md      # What you're building and why
  roadmap.md      # 7-day plan
  tech-stack.md   # Tools, frameworks, runtime
features/
  feature-1/
    requirements.md   # What the feature must do
    feature-plan.md   # How you'll build it
    validation.md     # How you'll verify it works
  feature-2/
  feature-3/
```

## How to use

1. Fill out `specs/` first — define the mission, roadmap, and tech stack before touching code.
2. For each feature, write `requirements.md` → `feature-plan.md` → `validation.md` before implementing.
3. Commit specs alongside code so reviewers can trace intent to implementation.
4. Add your source code at the repo root (or in a subdirectory of your choice).

## Submission checklist

- [ ] `specs/mission.md`, `roadmap.md`, `tech-stack.md` completed
- [ ] All features documented under `features/`
