## Area 5 — Easter Egg 2: Tap to Pet

**Status: Partial**

The decision logic (trust/willing evaluation, blocked conditions, stat mutations) lives entirely in `js/ui.js`, which the test runner does not load. That portion is outside the unit-test scope by architectural constraint — covered by manual testing only.

What is unit-testable (the `cat` object fields):

| Test name | File |
|---|---|
| "createCat feedCount defaults to 0" | cat.test.js |
| "createCat playCount defaults to 0" | cat.test.js |

Not unit-testable (manual only):

| Item | Reason |
|---|---|
| Trust condition: feedCount ≥ 1 AND playCount ≥ 1 | Logic in ui.js |
| Willing condition: trust + all stats > 75 | Logic in ui.js |
| Willing outcome: Happiness +10 | Applied inline in ui.js |
| Unwilling outcome: Happiness −15, Energy −10 | Applied inline in ui.js |
| Blocked when `cat.sick === true` | Guard in ui.js |
| Blocked when `cat.bellyActive === true` | Guard in ui.js |
